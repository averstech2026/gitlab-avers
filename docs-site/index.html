1

День 1–2

Sandbox-проект: создать и все клонируют

Создаёт Maintainer (A)

  1. Открыть группу AVERS
  2. Нажать + Новый проект
  3. Выбрать Создать пустой проект
  4. Имя: sandbox → Создать проект
  5. README лучше добавить позже (справа: Добавить README)

Клонируют все четверо

  1. Открыть проект (например LearnProject)
  2. Справа вверху кнопка Код ▼
  3. Вкладка HTTPS или SSH → скопировать URL
  4. В терминале: git clone …
  5. Проверить: у всех есть папка проекта

На пустом проекте URL также уже показан ниже на странице (блоки SSH / HTTPS). Кнопки «Новый файл», «Добавить README» — в правой колонке.

Правило: в main напрямую не пушим. Только через Merge Request.

2

День 3–4

Каждый делает 1 MR — друг друга ревьюите

Автор MR (каждый из 4)

  1. Ветка: git checkout -b feature/имя
  2. Правки → git commit → git push
  3. В GitLab: Создать merge request
  4. Или: Merge requests → New merge request
  5. Заголовок + описание → Create merge request

Кто кого ревьюит (по кругу)

A → ревьюит MR от B B → ревьюит MR от C C → ревьюит MR от D D → ревьюит MR от A
  • Открыть чужой MR → комментарии в коде
  • Готово: Approve / «Одобрить»
  • Maintainer: Merge / «Слить»

Идея дня: пройти полный круг — push, MR, комментарий, approve, merge. Тема может быть простой: файл notes/ваше-имя.md.

3

День 5

Issues → закрыть через MR

Завести 3–5 задач

  1. Issues → New issue
  2. Заголовок + описание → Create issue
  3. Назначить: поле Assignee
  4. Примеры: «.gitignore», «README», «правила команды»

Закрыть задачу через MR

  1. Ветка под задачу → коммит → push
  2. Create merge request
  3. В описании MR: Closes #12
  4. Ревью → Merge
  5. Issue закроется сама
Issue #12 → ветка → MR + Closes #12 → Merge → Issue закрыта ✓

После цикла — можно заводить боевой проект в группе AVERS (не sandbox).
